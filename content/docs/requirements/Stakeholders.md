# Stakeholders

This document defines the primary stakeholders of the system and their roles.  
It focuses on how each stakeholder interacts with the system, without describing requirements or expectations.

---

## 1. Individual Users

Entities representing real people.

### Role in the System
- Own identity and data  
- Create and update personal data by applications 
- Interact with other entities through applications  

---

## 2. Organizations and Communities

Entities representing groups of individuals.

### Role in the System
- Maintain group identity  
- Manage group-related data  
- Interact with other entities as a single unit  

---

## 3. Service Providers (Post-Platforms)

Entities providing applications and user-facing functionality.

### Role in the System
- Provide interfaces for interacting with the system  
- Compete by offering different functionality  

---

## 4. Infrastructure Operators

Entities operating system infrastructure components.

### Role in the System
- Store and serve data  
- Provide identity-related services  
- Ensure availability and performance of infrastructure  
- Execute system-level operations required for functioning  

---

## 5. Integrators and Developers

Entities extending or integrating with the system.

### Role in the System
- Build new services  
- Adapt existing systems to work with the ecosystem  
- Define data structures and mappings  
- Contribute to system evolution  

---

## 6. Regulators and External Authorities

External entities interacting with the system from outside.

### Role in the System
- Observe system activity where permitted  
- Influence constraints under which the system operates  

---

## 7. Ecosystem

The system as a network of interacting entities.

### Role in the System
- Enables interaction between independent participants  
- Maintains shared data space  
- Provide rools and standarts
